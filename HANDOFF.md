## 최근 세션에서 반영된 변경 (밸런스 v3)

사용자 피드백: ① 전체 점수가 너무 높다 ② 크게 오버슛하면 1고→2고가 공짜로 연쇄된다 ③ 족보에 안 들어간 카드까지 배수를 타는 게 이상하다.

1. **카드 칩 하향**: 광 15→12, 열끗 10→8, 띠 8→6, 쌍피 6→5, 피 3→2. 밤일낮장 보정 +3→+2. (이달의 패 ×2는 유지)
2. **고 목표 공식 교체** — 연쇄 고 구조적 차단:
   `goTarget(base, n, score) = max(ceil(base × (1+0.5n)), score + ceil(base × 0.25))`
   항상 현재 점수보다 크므로 고 선언 즉시 다음 단계가 충족될 수 없음. `chooseGo()`와 고 모달이 공유. 크게 넘긴 상태면 모달에 "(점수를 크게 넘겨 목표 상향!)" 표시.
3. **코어/플랫 스코어링** (족보 정합성):
   `detectHandInfo(cards)` → `{ handId, core }`. 점수 = `(족보칩 + core 카드 칩) × 배수 + 나머지 카드 칩(flat)`.
   - core 규칙: 오광/사광/삼광=광들, 총통/같은달3=그 달 카드, 고도리=새 3장, 단=띠 3장, 열끗셋/띠셋=해당 타입 전부, 피5=전부, 같은달2=짝 중 기본칩 합 높은 쪽, 무조합=전부(배수 1이라 무의미).
   - 조커 +칩은 기존대로 배수 타는 쪽(chips)에 가산. `computeScore` 반환에 `flat` 필드 추가됨 — UI 수식 표시는 `X칩 × Y배 + flat`.
4. **목표 커브 재조정**: 코어/플랫 변경으로 난이도가 폭등해서 TARGETS를 ~10% 하향: `[290, 430, 590, 790, 1030, 1350, 1800, 2350, 3050, 3950, 5100, 6600]`.

### B. 멍석(바닥패) 시스템 — 제안 확정, 미구현

- 판 시작 시 덱에서 3장 바닥 공개. 낸 카드와 같은 달 바닥 카드가 있으면 "짝!" 붙어서 그 내기에 합류 — 카드 칩 가산 + **족보 판정에도 포함**. 붙은 만큼 덱에서 바닥 보충.
- 확장: 바닥에 같은 달 2장일 때 붙이면 "따닥" ×1.5 배수.
- 구현 노트: `computeScore(cards, env, floorCards)`처럼 엔진 순수 함수 시그니처 확장, 프리뷰에서 붙을 바닥 카드 하이라이트. **밸런스 재시뮬레이션 필수** (실질 카드 수 증가 → 점수 상승).
- 네이밍은 고스톱/전통 용어 유지 (멍석, 따닥, 쪽 등).
